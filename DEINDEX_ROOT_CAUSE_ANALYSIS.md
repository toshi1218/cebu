# igrs.online デインデックス根本原因分析

## 概要

`site:igrs.online` で Google 検索結果が 0 件になった原因を調査した。
2026年2月初旬を境にインプレッション・クリック数がゼロに急落している。

## 結論（根本原因）

**2月7日〜9日の3日間で20回以上のリダイレクト関連デプロイが行われ、
リダイレクトループ（無限ループ）とURL構造の不安定性が発生したことが原因。**

Googlebot がクロール中にリダイレクトループに遭遇し、サイト全体の信頼性が低下。
結果として全ページがインデックスから除外された。

## Google が検出した問題（GSC レポートとの対応）

| GSC の問題 | 件数 | 原因 |
|-----------|------|------|
| **ページにリダイレクトあり** | 17件（検証失敗） | コンテンツページのURL構造が `.html` 有無で複数回変更。Google が以前のURLからリダイレクトを検出 |
| **リダイレクトエラー** | 15件（検証開始） | 2/7〜8 のリダイレクトループ。Googlebot がクロール時に無限ループに遭遇 |
| **見つかりません (404)** | 5件 | 削除されたページ（anshin-pack, audit_* 等）がインデックスに残存 |
| **robots.txt によりブロック** | 1件 | 一時的な設定ミス or Cloudflare のキャッシュの影響 |

## なぜ完全デインデックスに至ったか

1. **リダイレクトループ（最大の原因）**: 2/7〜8 にサイト全体がリダイレクトループ状態になった。Googlebot にとってサイトが「壊れている」状態。

2. **URL構造の不安定性**: 3日間で URL 構造が下記のように繰り返し変更:
   - `/page.html` → `/page` → `/page.html` → `/page`
   - middleware 有 → middleware 無 → middleware 有
   - `.html` ストリッピング 有効 → 無効 → 有効

3. **矛盾するリダイレクトシグナル**: `_redirects` ファイルと middleware が同時に存在し、競合するリダイレクトルールが発生。

4. **高頻度デプロイ**: 2/7〜9 の3日間で **30回以上のデプロイ**。Google のクロール周期（数時間〜数日）の中で URL 構造が何度も変わった。

## 現在のサイトの状態（2026/2/27 検証結果）

### 正常に動作している項目 ✅

| 項目 | 状態 |
|------|------|
| robots.txt | `Allow: /` + Sitemap リンク |
| sitemap.xml | 26 URL（全て正しい形式） |
| meta robots | コンテンツページ: `index,follow` / ユーティリティページ: `noindex,follow` |
| canonical タグ | 全ページに正しく設定 |
| `.html` ストリッピング | 301 → クリーン URL（正常動作） |
| www リダイレクト | `www` → `non-www`（301、正常動作） |
| レガシー URL リダイレクト | 全て 301 → 正しい宛先 |
| X-Robots-Tag | noindex ヘッダーなし（正常） |

### 残存する軽微な問題 ⚠️

1. **2ホップリダイレクトチェーン**: `www.igrs.online/page.html` → `igrs.online/page.html` → `igrs.online/page`（Cloudflare edge + middleware の2段階）
2. **クライアントサイド www リダイレクト**: `js/www-redirect.js` に JavaScript での www → non-www リダイレクトが残存（冗長）
3. **ルートと igrs-netlify 両方に設定ファイル**: `robots.txt`, `sitemap.xml`, `_redirects` がプロジェクトルートと `igrs-netlify/` の両方に存在

## 復旧に向けた推奨アクション

### 即座に実行すべきこと

1. **Google Search Console でインデックス登録をリクエスト**
   - 主要ページ（トップ、personal、cenomar 等）を個別に「URL検査」→「インデックス登録をリクエスト」
   - 1日あたりの上限があるため、重要度の高いページから順に実施

2. **sitemap.xml を GSC で再送信**
   - GSC → サイトマップ → `sitemap.xml` を削除して再追加

3. **今後は安定したデプロイを維持**
   - リダイレクト設定の変更は慎重にテストしてから本番反映
   - SEO に影響する変更は1日に1回以下に抑える

### 中期的な改善

4. **2ホップリダイレクトチェーンの解消**: middleware で www チェックを追加し、`www + .html` → `non-www clean URL` を1ホップで処理

5. **不要な `js/www-redirect.js` の削除**: Cloudflare edge + middleware で www リダイレクトは完結しているため

6. **設定ファイルの一元化**: ルートと igrs-netlify の重複ファイルを整理

## 回復見込み

現在の設定は技術的に正しい状態。Google がサイトを再クロール・再評価するまでに
**2〜4週間**かかる可能性がある。GSC の「検証開始」ステータスは Google が
再評価プロセスを開始したことを示しており、良い兆候。

安定した状態を維持し、GSC からインデックス登録リクエストを送信することが
最も重要なアクション。
